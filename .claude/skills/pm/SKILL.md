제안된 기능이나 변경사항을 사용자와 비즈니스 관점에서 평가하세요.

**기술 구현 방식, 코드 구조, API 설계는 언급하지 마세요. 오직 사용자 경험과 비즈니스 가치만 평가합니다.**

1. `docs/requirements.md`와 `docs/features.md`를 읽어 프로젝트 맥락 파악
2. 사용자가 설명한 기능 제안을 UX/비즈니스 관점에서 분석
3. 사용자 시나리오 완성도, 직관성, 일관성, 빠진 요구사항 평가
4. PM 관점의 피드백 출력
