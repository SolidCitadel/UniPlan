# 프로젝트 요구사항

## 1. 프로젝트 개요

**프로젝트명**: UniPlan - 시나리오 기반 대학생 수강신청 플래너

**목표**: 수강신청 실패에 대비하여, 사용자가 사전에 정의한 시나리오(Plan A/B/C)를 기반으로 실제 수강신청 시 빠르고 체계적인 의사결정을 지원하는 웹 애플리케이션.

**해결하고자 하는 문제**:

- **파편화된 계획**: 학생들이 여러 시간표 이미지를 만들지만, 유기적으로 연결된 시나리오가 아니어서 실제 상황에 적용하기 어려움
- **의사결정 병목**: 수강신청 실패 시 심리적 압박으로 즉석에서 최적 경로를 판단하기 어려움

## 2. 주요 기능 요구사항

### 구현 완료

1. **의사결정 트리 기반 시나리오 구축**
   - 기본 시간표 생성
   - 과목별 실패 가정 → 대체 시간표 시나리오 연결
   - 의사결정 트리 구조로 시나리오 관리

2. **실시간 수강신청 시나리오 네비게이터**
   - 수강신청 전용 UI
   - 과목별 성공/실패 입력
   - 실패 시 자동 대안 시나리오 전환

3. **다중 필터 강의 검색**
   - 과목명, 교수명, 학과, 캠퍼스 필터
   - 페이징 지원

### 미구현

4. **커뮤니티 강의평 연동** *(향후 확장)*
   - 에브리타임 등 강의평 정보 연동/링크

## 3. 기술 아키텍처

### 프론트엔드

- **프레임워크**: Next.js 15 (App Router)
- **상태관리**: React Query
- **UI**: shadcn/ui + Tailwind CSS

### 백엔드 (MSA)

- **API Gateway** (`api-gateway`)
  - 단일 진입점, JWT 인증/인가
  - Spring Cloud Gateway

- **개별 서비스**
  - `catalog-service`: 강의 목록 및 검색
  - `user-service`: 사용자 인증 및 계정
  - `planner-service`: 시간표, 시나리오, 수강신청

- **데이터베이스**: MySQL
- **서비스 간 통신**: REST API

## 4. 사용 시나리오 요약

1. **강의 검색** → 관심 과목 위시리스트에 추가
2. **시간표 생성** → 위시리스트에서 과목 선택하여 시간표 구성
3. **대안 시간표** → 특정 과목 실패 가정하여 대안 생성
4. **시나리오 트리** → 시간표들을 의사결정 트리로 연결
5. **수강신청 시뮬레이션** → 실시간 성공/실패 입력 → 자동 시나리오 전환
